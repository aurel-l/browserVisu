Data visualisation in the browser

on using D3.js or other libraries to interactively visualise data

Presentation URL: aurel-l.github.io/browserVisu

Why?

Problems it can potentially solve:

  • Interactivity
  • Easy and broad access
  • Easy deployement

access: need only a (recent) browser, no plugin

deployement: no logic server side

Drawbacks

  • No control on the browser/screen used
  • Different implementations, not the same support
  • Specific browser context (event loop, async, etc.)
  • Not easy to do advanced visualisation

Different browsers, old versions

Deep knowledge of how web technologies work

Easy to do things bad (not optimized, too much resource used, not reusable)

Libraries to the rescue!

D3.js

Data-Driven Documents

  • DOM manipulation (HTML and SVG)
  • data-binding between data structure and visual elements
  • Helper function to load data in various formats
  • Total freedom on the visual representation of the data
  • Good API documentation
  • No need for really deep understanding of SVG
  • Can be extend via plugins

BioJS

Represent biological data on the web

  • Set of visualisation tools for biological data
  • Made by biologists, for biologists
  • Repository of modules and components
  • Quite recents, few contributors, but active
  • Using recent/future features of the web

      • Potential use of a multiple sequence alignment element

      
<biojs-msa>
  <biojs-io-fasta url="./Q7T2N8.fasta" />
</biojs-msa>
